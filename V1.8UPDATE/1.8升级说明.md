六、数据备份与恢复
1. 备份MySQL数据库（最重要）
bash
# 备份到当前目录
docker exec guac-mysql mysqldump -u root -p'你的密码' guacamole_db > guacamole_backup_$(date +%Y%m%d).sql

# 压缩备份
docker exec guac-mysql mysqldump -u root -p'你的密码' guacamole_db | gzip > guacamole_backup_$(date +%Y%m%d).sql.gz
2. 恢复数据库
bash
# 从SQL文件恢复
docker exec -i guac-mysql mysql -u root -p'你的密码' guacamole_db < guacamole_backup.sql

# 从压缩文件恢复
gunzip < guacamole_backup.sql.gz | docker exec -i guac-mysql mysql -u root -p'你的密码' guacamole_db
3. 备份配置文件和应用
bash
# 备份整个部署目录
tar -czf vdi_deploy_backup_$(date +%Y%m%d).tar.gz /opt/vdi-deploy/

# 仅备份关键文件
mkdir -p /backup/vdi
cp /opt/vdi-deploy/config.env /backup/vdi/
cp /opt/vdi-deploy/vdi-web/app.py /backup/vdi/
cp -r /opt/vdi-deploy/vdi-web/templates /backup/vdi/
cp /opt/vdi-deploy/guacamole/docker-compose.yml /backup/vdi/
4. 恢复配置文件
bash
tar -xzf vdi_deploy_backup.tar.gz -C /
systemctl restart vdi-web
5. 设置定时备份（推荐）
bash
# 添加到 crontab，每天凌晨3点自动备份
crontab -e
# 添加以下行
0 3 * * * docker exec guac-mysql mysqldump -u root -p'你的密码' guacamole_db | gzip > /backup/guacamole_$(date +\%Y\%m\%d).sql.gz
七、升级过程错误总结
错误现象	原因	修复方式
申请提交500错误	get_db_connection 函数名不匹配	替换为实际函数名 get_db
审批创建失败	get_vm_ip(timeout=180) 参数名不匹配	改为 get_vm_ip(vmid, 180)
重复创建虚拟机	审批按钮可多次点击	前端按钮禁用+后端状态检查
审批504超时	Nginx默认超时60秒	延长至300秒
批量操作不显示	index.html未完整更新	直接替换完整HTML文件
定时任务弹窗失效	按钮事件冒泡触发折叠	添加 event.stopPropagation()
快照/修改按钮无反应	模板中事件绑定缺失	完整重写前端JS函数
八、注意事项
升级前务必备份数据库和应用文件

密码中避免使用特殊字符（如 $、!、#、...），建议使用纯字母数字

审批创建桌面时，系统会自动生成VMID（基于时间戳），避免手动指定冲突

使用统计功能需要桌面连接时记录数据，初期数据为空属于正常现象

批量删除操作不可逆，执行前请确认
